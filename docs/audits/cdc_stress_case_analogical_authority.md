## Why Analogy Is Dangerous

**Most people either:**
1. **Worship analogies** - "Like evolution, so it must work!"
2. **Outlaw analogies** - "Never compare to other fields!"

**Both wrong.**

**Correct approach: Domesticate analogies**
- Use safely (mechanical mapping + breaks)
- Use reluctantly (only when aids understanding)
- Never use for validation/proof/authority

**Analogy launders authority by:**
1. Transferring credibility from established field
2. Suggesting inevitability (evolution = natural/correct)
3. Implying validation (law = legitimate/binding)
4. Creating false equivalence (this = that, therefore properties transfer)

**CDC blocks this by requiring:**
- Mechanical mapping (no vibes)
- Declared breaks (no false equivalence)
- Explicit denial (no authority transfer)
